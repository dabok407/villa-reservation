- SSH 접속 OK (`ec2-user`, 키 기반). **Java 17**(Amazon Corretto) — Spring Boot 2.7 jar 호환 구동.
- `free -h` → available **416Mi, swap 0** (전일과 동일, 여전히 빠듯).
- **발견: villa 예약 앱이 이미 구동 중**(`~/villa-reservation/app.jar`, `-Xmx384m -Dspring.profiles.active=prod`, 5월 중순부터). 코인봇도 동시 구동.
- 기동 스크립트(`script/startup.sh`)는 `-Xmx256m -jar app.jar`로, **실제 실행 명령과 불일치**(실행 중 프로세스엔 prod 프로필·관리자 인자 포함). app.pid도 stale. → startup.sh를 신뢰하지 않고, 재시작 위험 회피 위해 **기동 명령은 미수정**.

### 보안 관찰 (수정 권고)
- 실행 명령에 **관리자 자격증명이 `-D` JVM 인자로 평문 노출**(`ps`로 보임). → 환경변수/600 파일로 옮길 것을 권고. (AI 키는 이 실수를 반복하지 않도록 처음부터 환경변수로 설계)

## 2026-06-02 · AI 키 주입 메커니즘 구성 (방식 B = 환경변수)
- `~/.villa.env` 생성, 권한 **600**(소유자 전용). 키는 이 파일에만(커밋·`-D` 인자 금지).
- `~/.bash_profile`에서 로드(`set -a; . ~/.villa.env; set +a`) → 로그인 셸에서 기동 시 앱 프로세스가 환경변수 상속. **키가 `ps`에 노출되지 않음.**
- 멱등 구성(재실행 안전), 로드 테스트 통과. 실제 키 값은 운영자가 별도 입력(전송 경로에 비밀 미노출).
- 적용 시점: 신규 AI 빌드 배포·재기동 시 발효(현재 구동 중인 구버전 jar는 AI 기능 없음).
