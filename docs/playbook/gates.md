# 결정 게이트 3종 (Decision Gates) — 채움 템플릿

> 코드를 더 짜기 전에 **멈춰서** 확인하는 3개 관문. 각 게이트는 "통과/보류"를 판정한다.
> 라이브코딩에선 이 항목들을 *소리내어* 짚는다. 준비 단계에선 빈칸을 실제로 채운다.
> 틀리면 비싸게 되돌려야 하는 지점에서만 멈춘다(매 파일 X).

---

## 🚦 게이트 1 — 범위 (Scope)  [P1 종료 시]

```
■ 한 문장 정의:        (무엇을/왜)
■ MVP에 포함:          - 
                       - 
■ 명시적 제외(가드):    - (이번에 안 하는 것 — 나중에 "왜 안 했나" 방어)
                       - 
■ 완료 정의(DoD):       (측정 가능하게)
■ 일정/제약:            (시간, 리소스, 의존성)
■ 최대 리스크 1개:      (무엇이 틀어지면 가장 아픈가)
```
**통과 기준**: 제외 목록이 비어있지 않다(= 범위를 의식적으로 좁혔다). 한 문장으로 설명된다.

---

## 🚦 게이트 2 — 설계 (Design)  [P2 종료 시]

먼저 [../dev-checklist.md](../dev-checklist.md) 0~10번을 훑고, 해당 항목만 아래에 옮긴다.
```
■ 핵심 불변식:          (절대 깨지면 안 되는 규칙 — 예: 충돌=반개구간)
   └ 단일 진실 원천:     (어디서 보장? 중복 구현 위험은?)
■ 기존 재사용 vs 신규:   (재사용할 것 / 새로 만들 것 — 복제 금지)
■ 최소 침습 삽입 지점:   (기존 코드 수정 최소화 — 어디에 붙이나)
■ 인터페이스 형태:       동기REST □  배치/스케줄러 □  실시간(WS/SSE) □  비동기/MQ □
■ 데이터/트랜잭션:       (읽기전용? 쓰기 경계? 실데이터 안전?)
■ 보안:                 (인증/인가, 비밀정보, 입력검증, 외부입력 신뢰경계)
■ 실패 모드:            (외부 실패 시 fallback, 타임아웃, 멱등성)
■ 되돌리기 비용:         (이 설계가 틀리면 얼마나 비싼가)
```
**통과 기준**: 불변식이 단일 원천에서 보장된다. 기존 구조를 깨지 않는다. 체크리스트 해당 항목에 답이 있다.

---

## 🚦 게이트 3 — 검증 (Verify)  [P4 종료 시]

```
■ 엣지케이스 목록:       - (경계값: 동일값, off-by-one, 빈값, 음수, 시간대)
                       - (상태 누수: 취소/완료 데이터)
                       - (AI: 환각, 오파싱, 모호 입력)
■ 테스트 결과:          단위 □  통합 □  E2E □   (N/N PASS)
■ 회귀:                (기존 기능 안 깨졌나 — 전체 테스트 통과?)
■ 검증 로그 기록:        (입력/기대/실제/판정/조치 — verification-log.md)
■ 정직한 한계:          (아직 검증 못 한 것, 알려진 제약)
```
**통과 기준**: 핵심 불변식이 테스트로 고정됐다. FAIL은 숨기지 않고 원인·수정까지 기록됐다.

---

### 사용 팁
- 게이트는 **승인 도장이 아니라 자기 점검**이다. 준비 땐 채우고, 면접 땐 말한다.
- 게이트에서 "보류"가 나오면 이전 페이즈로 되돌아간다(이게 정상).
- 3개 게이트만 지켜도, 범위 폭주·구조 파괴·검증 누락이라는 3대 사고를 막는다.
